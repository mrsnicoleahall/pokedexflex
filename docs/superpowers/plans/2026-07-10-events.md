# PokeDexFlex Events Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add event Pokémon to PokeDexFlex as a browsable reference catalog of real historical distributions, plus a small set of sample owned event specimens in a demo collection.

**Architecture:** A new `events` reference table (seeded from a curated, committed JSON of well-documented distributions), a browse API, and a catalog page — mirroring the proven species pattern (schema → snapshot JSON → generated seed SQL → Hono route → React page). Sample ownership is demonstrated via a seeded demo user + box + event specimens, with an endpoint that marks which catalog events the demo collection owns. True per-user check-off is deferred to the Auth plan.

**Tech Stack:** Cloudflare Workers, D1, Drizzle, Hono; Vite/React/TS; Vitest.

## Global Constraints

- Cloudflare Workers runtime (no Node-only APIs in Worker code); TypeScript strict; typecheck of record is `tsc -b`.
- Database via D1; ALL schema changes through drizzle-kit migrations. Seed is DML only.
- **Data integrity: do NOT fabricate event data.** Only seed fields that are well-documented for a given event. Leave uncertain fields (`ot_name`, `ot_id`, exact `ribbons`) NULL rather than guessing. Every seeded event must be a real, verifiable distribution. Keep the starter set modest (~15–20 events) and mark it as a starter set.
- Reuse existing patterns: build-time script under `scripts/` (in the `tsc -b` graph, guarded with the `isRunAsEntryScript()` entry pattern); `INSERT OR REPLACE` idempotent seed with a UNIQUE conflict target; forms/events already first-class in the `specimens` table.
- The `events` catalog links to species/forms already in the DB.

## Data Model

**events** (reference/curated)
- `id` (int PK autoincrement)
- `slug` (text, UNIQUE) — stable identifier, e.g. `magearna-qr-2016`
- `name` (text) — display name, e.g. "Magearna (QR Code)"
- `speciesId` (int FK→species.id)
- `formId` (int FK→forms.id, nullable)
- `year` (int, nullable)
- `games` (text, nullable) — e.g. "SM/USUM"
- `region` (text, nullable) — e.g. "International", "NA", "JP", "EU"
- `method` (text, nullable) — distribution method, e.g. "QR Code", "Serial Code", "Local Wireless", "Nintendo Network"
- `otName` (text, nullable), `otId` (text, nullable) — ONLY if the event fixes them; else NULL
- `ribbon` (text, nullable) — notable ribbon if well-documented (e.g. "Classic")
- `isShiny` (int bool, default 0)
- `notes` (text, nullable)

UNIQUE(slug) is the `INSERT OR REPLACE` conflict target (idempotent re-seed).

Demo ownership uses existing tables: a seeded `users` row (`id="demo"`), a `boxes` row ("Events"), and a handful of `specimens` rows with `isEvent=1`, `eventName` set to the event `name`, `source="manual"`, linked to the demo user.

- [ ] **Step 1: Write failing test** — in `tests/db/schema.test.ts`, insert a species (id 801), then an `events` row (slug `magearna-qr-2016`, name, speciesId 801, method "QR Code"); read back; assert slug + method. Add a second insert with the SAME slug via `INSERT OR REPLACE` semantics (Drizzle `.onConflictDoUpdate({ target: events.slug, set: {...} })`) and assert the row count for that slug stays 1.

```ts
// tests/db/schema.test.ts (add)
import { events } from "../../src/db/schema";
it("events: insert, read, and slug is unique (idempotent upsert)", async () => {
  const db = getDb(env.DB);
  await db.insert(species).values({ id: 801, name: "magearna", generation: 7, types: "[]" });
  await db.insert(events).values({ slug: "magearna-qr-2016", name: "Magearna (QR Code)", speciesId: 801, method: "QR Code", isShiny: 0 });
  await db.insert(events).values({ slug: "magearna-qr-2016", name: "Magearna (QR Code) v2", speciesId: 801, method: "QR Code", isShiny: 0 })
    .onConflictDoUpdate({ target: events.slug, set: { name: "Magearna (QR Code) v2" } });
  const rows = await db.select().from(events).where(eq(events.slug, "magearna-qr-2016"));
  expect(rows).toHaveLength(1);
  expect(rows[0].name).toBe("Magearna (QR Code) v2");
});
```
